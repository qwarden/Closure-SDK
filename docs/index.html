Closure

Closure is a geometric computer — Von Neumann architecture, but built on geometry instead of binary strings. Data is ordered hierarchically in space rather than addressed as bit patterns, and that changes what a computer can be.

Because data already lives in geometry, memory, verification, and execution share the same underlying structure. That same structure is also a new transformer architecture: memory, retrieval, and model collapse into one object. A new general computer and a new transformer architecture are the same thing.

The sphere to the right is that geometry running live. Every point is a record, σ is the drift of the whole sequence. Pull to open it, release to close.

Memory

The same primitive that represents a record also checks whether a sequence of them is coherent. The database is not separate from the integrity layer — they are the same object. Point lookups 6.9× faster than SQLite. Integrity checks 89,885× faster. Details →

Verification

Finding corruption normally means scanning. Here the sequence measures its own coherence — one number tells you what broke, where, and how much. 50 corruptions across 100,000 records: 384 ms. A Merkle tree takes 18.6 seconds. Details →

Computation

Programs and data share the same geometric representation. The learning system that runs without parameters is the same architecture turned toward execution — no weights, no gradient descent, no separation between what is stored and what runs. Watch it live →
