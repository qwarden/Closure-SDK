Data Verification

Know whether your data arrived intact — and exactly what didn’t

The core primitive. Every record gets hashed and composed onto S³. When a sequence is coherent, the composition closes back to where it started. When it isn’t, the product drifts, and the drift is exact: how much, where in the sequence, and what kind of fault — missing record or reorder, always distinguishable.

Two tools implement this. The CLI handles file-to-file comparison and live stream monitoring. The bind() / prism functions handle zero-knowledge verification: two parties confirm they hold the same data by exchanging 16 bytes. The raw data never moves. Neither side can fake agreement.

A Merkle tree must rebuild after every fault it finds — O(k·n) for k faults. This system composes both sequences once and classifies all faults in a single walk. The cost is O(n) regardless of fault count.

Geometric Database

Storage with integrity built in

Closure DNA is a database where every column is a composition tree on S³. The data lives in the structure, not alongside it. Integrity is not an optional check you run afterward — it is native to how the data is held. Any record change, deletion, or insertion is detectable in microseconds.

The sidecars are the key architectural point. Your raw data lives in one place. The geometric state — the verification layer — is a derived artifact, rebuildable from the columns. Two systems that want to confirm they hold identical data exchange a single composition product: one point on S³, 32 bytes. Neither side exposes its records.

Learning Architecture

Computation without parameters

Enkidu is a prototype agent that forages, builds shelter, and survives in a hostile environment. It has no neural network, no trained weights, no learned parameters whatsoever. Every decision is a single geometric comparison: which distance from identity is larger, and which action most reduces it.

The environment computes the gradient. Identity is homeostasis. Free energy is geodesic distance from identity on S³. The geometry supplies the path home. Everything you see in the demo — the foraging routes, the hesitation when drives are balanced, the camps extending his territory in scarce environments — emerges from that one operation applied repeatedly to a world that refuses to leave him at rest.

Why this matters beyond the demo

A transformer learns attention through millions of parameters trained on billions of tokens. Friston’s Free Energy Principle describes the same process through variational calculus. Enkidu computes both directly: compose, measure distance from identity, act on the largest. The correspondence is not metaphorical — it is exact.

Memory, verification, and execution share one representation. The database is not a separate layer from the compute. This is the direction of the work.
